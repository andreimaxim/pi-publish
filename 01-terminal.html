❯ Pi stores sessions in a tree-like format, right?
The user is asking about Pi's session format. I already read the session.md docs earlier in this conversation, so I can answer from what I know.
Yes. The session file is JSONL where each entry has an id and parentId, forming a tree. The first entry has parentId: null, and every subsequent entry points to its parent. Branching (via /tree) creates a new child off an earlier node.

❯ Can you show me what the session parser would look like in Ruby?
The user wants to see a Ruby version of the session parser. I'll write a clean class that reads JSONL, builds the tree, and walks from leaf to root.
Here's a minimal session parser: 
# frozen_string_literal: true

require "json"

class SessionParser
  Entry = Struct.new(:id, :parent_id, :type, :data)

  attr_reader :header, :entries

  def initialize(path)
    @entries = {}
    @header = nil
    @leaf_id = nil

    File.foreach(path) do |line|
      raw = JSON.parse(line, symbolize_names: true)

      if raw[:type] == "session"
        @header = raw
        next
      end

      entry = Entry.new(raw[:id], raw[:parentId], raw[:type], raw)
      @entries[entry.id] = entry
      @leaf_id = entry.id
    end
  end

  def branch(from_id = @leaf_id)
    result = []
    current = @entries[from_id]

    while current
      result.unshift(current)
      current = @entries[current.parent_id]
    end

    result
  end

  def messages
    branch.select { |e| e.type == "message" }
           .map { |e| e.data[:message] }
  end
end
It reads the JSONL line by line, stores entries in a hash by ID, and branch walks from the leaf back to root via parent_id. The messages method filters to just message entries.
